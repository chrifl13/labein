Kontrollspørsmål 

0.	Javascript kan sees på som en tilleggsfunksjon til HTML som gjør det mulig for oss å lage klient-side funskjonalitet på web-siden vår. Javascript kan brukes til å, lokalt, gjøre enkle, eller mer avanserte operasjoner, eller forandre brukeropplevelsen på siden.
1.	CSS id og Class elementer funger på ulikt vist. CSS ID sine «elementer» er blant annet unike. Elementene har alle sin egen spesielle id, og disse elementene kan kun benyttes en gang per side ettersom «ID» en er reservert til det tilfeldige elementet per side. Dersom en prøve å benytte seg av elementer med samme id vil en få feilmelding.  Derimot elementer med samme «Class» kan benyttes flere ganger per side  eller motsatt at du bruker flere «class» på det samme elementet. 
2.  Brukbarhet mellom de to «listene» kan debatteres.  Lista i deloppgave 0 er mer enkelt men kan til tider være rotete, selv med sortering etter farge/bokstaver o.l. Listen i oppgave 1 er enklere å holde styr på men kan også være litt for vanskelig å navigere gjennom. Lista i oppgave 0 kan også tilby enkel navigering via «ctrl-f» kommando som kan gjøre det enklere.  En måte å sortere innholdet i listene etter visse kategorier ville vært det beste.
3. Wikipedia løser problemstillingen med en lang liste av programmspråk ved at Wikipedia hendviser til hotlinker som lett kan navigeres gjennom via at det er mulig å lete gjennom språkene i en oversiktlig og alfabetisk metode som gjøre det lett å finne frem. 
4. Forskjellene mellom de to er at <span>er et inline-element, hvor div er et block-element. Div's brukes hovedsakelig til å lage "bokser". Span kan man si skal brukes inni andre elementer for å gjøre noe spesielt med deler av innholdet, gå mellom tekst linjer o.l. 

5.  Brukbarhetsproblemer som kan oppstå ved implementerinen av deloppgave 2 kan være at blant annet, det vises feil respons i feltet hvor brukeren skal bli obs på hva som har blitt selektert. Innholdet i dropdownboksen kan vise feil/overlappe og dermed skape en uoversiktlig meny.
